# 01 — Database Migrations & Query Helpers

## Goal

Generate the initial Drizzle migration from the existing schema and create a reusable `notDeleted()` filter helper so all queries consistently exclude soft-deleted rows without repeating the same `WHERE` clause.

## Context

The database schema in `packages/database/src/schema.ts` is fully defined (4 tables, 3 enums, conditional unique indexes) but no migrations have been generated yet. The `db:generate` script exists in root `package.json` but has never been run. The schema docs require that queries filter `WHERE deleted_at IS NULL` by default — a shared Drizzle filter helper will enforce this.

## Files to Create

- `packages/database/drizzle/0000_initial_migration.sql` — generated by `bun run db:generate`
- `packages/database/src/filters.ts` — `notDeleted()` helper function for each table

## Files to Modify

- `packages/database/src/index.ts` — re-export `notDeleted` from filters

## Files to Reference

- `packages/database/src/schema.ts` — source schema for migration generation
- `packages/database/drizzle.config.ts` — existing Drizzle config
- `project-wiki/database/conventions.md` — soft-delete query conventions

## Acceptance Criteria

- `bun run db:generate` produces a valid initial migration
- `notDeleted()` helper works for all 4 tables (themes, artists, socialConnections, content)
- All exports from `packages/database` are accessible via the package's main entry point
- `bun run validate` passes with zero errors
